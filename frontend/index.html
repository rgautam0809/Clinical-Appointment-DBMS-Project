Time Conflict Detection Demo

This section demonstrates the database's ability to detect and prevent time conflicts.

How It Works:

  • ✅ Database Triggers: Automatically prevent double booking at the database level
  • ✅ Stored Procedures: Check for time slot overlaps before booking
  • ✅ Backend Validation: API checks for conflicts before inserting/updating
  • ✅ Frontend Alerts: Real-time conflict notifications to users

Conflict Scenarios Tested:

  1. Attempting to book same doctor at overlapping times
  2. Updating appointment to a conflicting time slot
  3. Multiple patients booking same slot (only first succeeds)
  4. Cancelled appointments freeing up time slots
